Gem Jam

Gem Jam

I love the first time of exploring an open game world. From snowy mountains to arid deserts, there's a lonely enjoyment to be had in every new experience. I wanted to use Javascript combined with old-school pixel graphics to create a miniaturized version of this feeling.

gemJam is a browser based game built using the Phaser.io framework and HTML5. Exploring a 2D platform-based world, the player must locate gems while navigating dangerous pitfalls and traps. The only way to win is to explore. To add some sense of competition, the game features a timer as well as a score counter to compare previous attempts.

I used the application Tiled to help me in generating a large JSON-based side scrolling worldmap using an open source, small-sized spritesheet to allow for quick loading times. The game uses states to keep the code organized and easy to use. All the assets are preloaded for quickness once the actual game has begun, however due to the size of the map it does get the occassional glitch of having the player sprite stuck in the middle of a block.

Mind Dump

Mind Dump

I initially wanted to create a private website where a user could do a daily "mind dump" exercise a la morning pages, adding Watson into the mix to analyze the text as a whole. I wanted a private space to save and track these daily writings, almost like a journal. The UI would be clean and minimal to prevent distraction and allow for a mobile/tablet friendly experiment to write and analyze on the go (for example, while commuting on the bus).

In addition to boosting creativity through stream-of-consciousness writing, Mind Dump utilizes IBM's Watson platform to perform a 'tone check' to provide insights about three types of tones in a piece of text including emotion (negative emotions, cheerfulness, anger), social propensities (openness, agreeableness, and conscientiousness), and writing styles (analytical, confident, and tentative).

The end goal is to provide data-driven insignt into one's inner thoughts to better prioritize, synchronize, and live a more mindful day.

Accio Bike

Accio Bike

Seattle has one of the highest bike theft rates in the nation with over 1,550 reported stolen last year alone. Many end up being sold or bartered as currency on the street. For a savvy thief, stealing a bike is easy. For a victim, getting a bike back is hard.

Accio Bike is a registry for your bicycle to proactively prevent and recover thefts. Without creating an account, a user can search the database of missing bikes for information like a bicycle's unique serial number, brand, model, color, size, or the zip code from which it went missing. If your bike has gone missing, create a free account and fill in your bike's information. The app data scrapes common sites to sell stolen goods and matches them to your missing bike's profile.

Completed in a one-week sprint, Accio Bike was a group collaboration using Ruby on Rails for the web framework, PostgreSQL for the database, and Git for version control. I was acting project manager and front-end developer. Whiteboarding, wireframes, ERD diagrams, and clearly outlining our CRUD functionality kept the project organized and merge conflicts minimal, as everyone had a clearly delegated job. We separated components into self-contained partials which allowed group members to tackle features displayed on the same page. Keeping organized with a clear vision of our minimum viable product allowed us to quickly and efficiently dial in our required features and move on to the fun stuff. We had time to try our hands at Cloudinary image uploads, interactive jQuery forms, Nokogiri data scraping, SVG/CSS animations, and a hand-coded search function.

Birbchat

When I was first exposed to the web, discovering chatrooms and clients came soon afterwards. I would spend hours chatting to friends in all areas of the globe. I never had a phone line going to my home growing up and the internet was my next best option. Instant messaging had a profound effect on me and I decided to make my own tiny version, now that I had the tools and knowledge.
